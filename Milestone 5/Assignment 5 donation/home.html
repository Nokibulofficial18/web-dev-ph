Frequently Asked Questions

What is Document Object Model (DOM)?

The DOM (Document Object Model) is a programming interface that represents the structure of a webpage as a tree of objects. It allows scripts (like JavaScript) to dynamically change, update, and interact with the content and layout of the page without reloading it.

How do you select an element from the DOM?

To select an element from the DOM in JavaScript, I can use:
By ID: document.getElementById("id")
By class: document.getElementsByClassName("class")
By tag name: document.getElementsByTagName("tag")
By CSS selector: document.querySelector("selector") (for one) or document.querySelectorAll("selector") (for all)
These let me target and manipulate elements in the DOM.

What is event delegation in the context of the DOM, and why is it useful?

**Event delegation** is a technique where a single event listener is attached to a parent element to manage events from its child elements. This leverages event bubbling, reducing the need for multiple event listeners.
### Benefits:
- **Better performance** (fewer event listeners).
- **Works with dynamically added elements.**
- **Simplifies code.** Example: One listener on a parent handles clicks from all its children.

How do you manipulate an element's attributes and styles using the DOM?

To manipulate an element's attributes and styles in the DOM:
### **Attributes:**
- **Get:** `element.getAttribute("attributeName")`
- **Set:** `element.setAttribute("attributeName", "value")`
- **Remove:** `element.removeAttribute("attributeName")`
### **Styles:**
- **Set style:** `element.style.propertyName = "value"` (e.g., `element.style.color = "blue"`)
Example: One listener on a parent handles clicks from all its children.
